2. Automatización y procesamiento ETL de noticias en la web

Tecnologías aplicadas: Python, Web Scraping, Selenium, BeautifulSoup, Power Query, Lenguaje M.
▼

Implementé un proceso de ETL para recopilar y estructurar noticias desde un sitio web, optimizando su análisis y uso por parte de los interesados. Este proyecto transformó un proceso manual y tedioso en uno automatizado, eficiente y escalable.

En principio, el equipo realizaba este proceso de manera manual, lo que implicaba navegar por cada sección del sitio web, identificar información relevante y estructurarla manualmente. Este enfoque era lento, propenso a errores y limitaba la capacidad del equipo para analizar grandes volúmenes de datos.

Funcionalidades claves:

  • Extracción de Datos: Uso de Selenium y BeautifulSoup para realizar web scraping.
  • Almacenamiento: Generación de un archivo TXT con toda la información para su procesamiento posterior.
  • Transformación de Datos: Estructuración y limpieza de datos con Power Query.
  • Enriquecimiento de Datos: Identificación de palabras clave, fechas, enlaces y frases relevantes.
  • Automatización Semanal: Ejecución automática del script de Python cada semana.

Impacto:

  • Automatización del proceso de recopilación de noticias.
  • Reducción del tiempo de búsqueda y estructuración de información.
  • Facilitación del análisis de identificación de frases y palabras claves.

Conclusión:

  • Este proyecto demostró cómo la automatización de procesos ETL puede transformar la manera en que las organizaciones recopilan y analizan información. Al implementar un sistema automatizado para la extracción y transformación de noticias, el equipo logró reducir el tiempo de recopilación, mejorar la calidad de los datos y detectar oportunidades clave de manera más eficiente.

from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from bs4 import BeautifulSoup
from time import gmtime, strftime
import time

url = "https://paginawebparascraping.com/"

# Configuración del driver
driver = webdriver.Chrome()
driver.get(url)

# Esperar que la página cargue completamente
time.sleep(10)

# Realizar scroll hacia abajo
scroll_pause_time = 5  # Tiempo de espera entre scrolls

# Hacer desplazamientos hacia abajo
for _ in range(20):
    # Hacer scroll hacia abajo
    driver.execute_script("window.scrollTo(0, document.body.scrollHeight);")
    
    # Esperar para que se carguen nuevos elementos
    time.sleep(scroll_pause_time)

# Extracción de info
soup = BeautifulSoup(driver.page_source, "html.parser")
articles = soup.find_all("article")
print(f"Found {len(articles)} articles")  # Confirmar cantidad de artículos

scraped_data = []
for article in articles:
    # Extraer los datos
    link_tag = article.find("a", class_="bodytext content")
    link = link_tag["href"] if link_tag else "No link found"

    title_tag = article.find("h2")
    title = title_tag.get_text(strip=True) if title_tag else "No title found"

    date_tag = article.find("time")
    date = date_tag["datetime"] if date_tag and date_tag.has_attr("datetime") else "No date found"

    # Imprimir datos para depuración
    print(f"Link: {link}")
    print(f"Title: {title}")
    print(f"Date: {date}")

    # Agregar a la lista solo si todos los datos están disponibles
    if link != "No link found" and title != "No title found" and date != "No date found":
        scraped_data.append({"title": title, "link": link, "date": date})

# Guardar los datos en un archivo .txt
fecha = strftime("%Y-%m-%d %H-%M-%S", gmtime())
file_name = f"file_scraped_data_{fecha}.txt"

with open(file_name, "w", encoding="utf-8") as file:
    for entry in scraped_data:
        file.write(f"Title: {entry['title']}\n")
        file.write(f"Date: {entry['date']}\n")
        file.write(f"Link: {entry['link']}\n")
        file.write("\n")  # Línea vacía entre noticias

# Mensaje final
print(f"Datos guardados correctamente en '{file_name}'")

# Cerrar el navegador
driver.quit()

3. Análisis y Clasificación de Vinos utilizando Modelos de Machine Learning

Tecnologías aplicadas: Python, Regresión Logística, K-Nearest Neighbors (KNN), Máquinas de Vector Soporte (SVM), Scikit-Learn, Matplotlib, numpy.
▼

En este proyecto, desarrollé y evalué modelos de Machine Learning para clasificar vinos en tres categorías basándome en 13 atributos numéricos. Utilicé técnicas de preprocesamiento, como la normalización de datos, y comparé el rendimiento de modelos de Regresión Logística, KNN y SVM..

Objetivo:

  • El objetivo fue predecir la clase de vino utilizando el conjunto de datos "Wine Recognition Dataset" de Scikit-Learn, logrando precisiones superiores al 95%.

Metodología:

  • Carga y Preprocesamiento: Normalicé los datos para mejorar el rendimiento de los modelos.
  • División de Datos: Separé los datos en conjuntos de entrenamiento (75%) y prueba (25%).
  • Modelos Evaluados: Regresión Logística, KNN y SVM..
  • Evaluación: Utilicé precisión (accuracy) y matrices de confusión para comparar los modelos.

Impacto:

  • Preprocesamiento de datos, reducción de dimensionalidad y clustering no supervisado.
  • Extracción de patrones en conjuntos de datos sin etiquetas, aplicable a diversos sectores como segmentación de clientes, análisis de productos entre otros.

Resultados:

  • Regresión Logística: 98% de precisión con datos normalizados.
  • KNN: 96% de precisión con datos normalizados.
  • SVM: 98% de precisión con datos originales y normalizados.

Conclusión:

  • Los modelos de Regresión Logística y SVM mostraron el mejor rendimiento, con una precisión del 98%. El KNN con datos normalizados también fue efectivo, aunque con una precisión ligeramente inferior (96%). Este proyecto demostró la importancia del preprocesamiento de datos y la selección adecuada de algoritmos para tareas de clasificación.
